Design System & Aesthetic Language
The Color Palette
To establish clinical trust and a modern tech feel, use a clean, high-contrast palette with strong indicators for validation and safety states:

Primary Canvas Background: A crisp, stark white or a very soft, clinical light gray for maximum readability.

Text & Deep Hierarchy: A rich midnight navy or deep charcoal for primary text, avoiding harsh pure blacks.

Trust & Accent Colors: A reassuring medical teal or deep sapphire blue for buttons and primary actions.

The "Zero-Hallucination" Stamp: A vibrant emerald green used strictly for verified nodes, safety badges, and successful matches.

Typography & Spatial Feel
Typeface: A clean, geometric sans-serif (like Inter or Plus Jakarta Sans).

Spacing: Generous white space around containers to prevent cognitive fatigue—rare disease data is complex, so the surrounding layout must feel breathing and light.

 Homepage Structural Blueprint (Section by Section)
1. Navigation Header
A minimal, fixed top bar that stays out of the way.

Left side: OrphanLink logo in a bold navy font with a small green dot representing an active, validated system state.

Right side: Clean navigation links (Technology, Security & Privacy, For Clinicians) ending with a primary button: Launch Portal.

2. The Hero Section (The Hook Line)
This is the first element a user sees. It must declare exactly what the platform does and why it is bulletproof.

The Main Hook Line: > "Autonomous Trial Matching. Zero Hallucinations."

The Supporting Value Proposition: > "OrphanLink uses strict state-machine orchestration to parse complex patient biomarkers and match them against live data from ClinicalTrials.gov with absolute context fidelity. No guessing. No false hope."

Data Verification Subtext: A small, muted gray row of text underneath reading: Fed continuously via official RxNorm, MyGene.info, and ClinicalTrials.gov APIs.

3. The Dual-Gateway Actions (The Big Choices)
Directly beneath the hero text sit two massive, vertically stacked containers. These are highly tactile cards that invite interaction.

Option A: The Automated Report Vault (Card)

Visuals: A clean dashed dropzone box with an icon of a document.

Text: "Upload Biomarker Report" (Supports PDF, JSON, TXT).

Subtext: "Our automated pipeline triggers intelligent OCR fallback for scanned papers to isolate variants near-instantly."

Option B: The Conversational Coordinator (Card)

Visuals: A modern chat bubble icon with an indicator reading "Agent Active".

Text: "Consult Clinical Intake Coordinator"

Subtext: "Don't have a report handy? Speak directly with our guided agent to step through your current diagnosis and active therapies."

 The Interaction Flow: What Happens On Click?
When a user interacts with either option, the homepage transitions entirely into a focused application environment. Here is exactly how those windows should look and behave:

Flow A: Clicking "Upload Biomarker Report"
The Trigger: The user drags a file into the dropzone or clicks the card to select a file.

The Transition: The homepage smoothly fades out. A new full-screen interface expands cleanly from the center.

The New Window Layout:

Left Column (The Pipeline Tracker): A vertical timeline appears, showing the real-time execution steps. It immediately lights up the first bubble: [1] Extracting & Parsing Text via PyMuPDF. If text density is low, it flashes a warning and switches to [1] Initializing Tesseract OCR Fallback.

Right Column (The Progress Board): A spinning indicator shows that the backend is abstracting data into your strict JSON schema. Once done, the timeline lights up [2] Standardizing via RxNorm/MyGene.

The Result Screen: Once all backend nodes pass (including the Validator Node's safety loop), the screen splits into a clean presentation dashboard showing matched trials on the left and the interactive patient action checklist on the right.

Flow B: Clicking "Consult Clinical Intake Coordinator"
The Trigger: The user clicks the conversational card.

The Transition: The landing page elements recede smoothly into the background, and a dedicated, split-pane chat sanctuary slides into view.

The New Window Layout:

Main Focus Area: A centered, highly legible chat conversation interface. The Conversational Agent Node introduces itself with your prompt instructions: "Welcome to OrphanLink. I am your clinical intake assistant. To help find matches, could you start by telling me your age?"

Subtle Bottom Status Tracker: A horizontal status bar quietly keeps track of what information has been secured so far: [Age: Empty] [Mutations: Empty] [Therapies: Empty].

The Result Trigger: As the user answers, the text tokens are extracted live, and the status bar updates to green icons. The moment the user provides the final piece of info, the agent prints its concluding message, the hidden token [EXTRACTION_COMPLETE] fires to the backend, and the view smoothly transitions into the active matching pipeline, showing the live verification steps before outputting the final trial list.
